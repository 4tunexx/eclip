1. PRODUCT OVERVIEW
1.1 What is Eclip.pro?

Eclip.pro is a third-party competitive platform for Counter-Strike 2 (CS2), similar in spirit to FACEIT/ESEA, but:

With a modern neon/green design and clean UI.

With transparent progression (XP, levels, ranks, missions).

With a soft currency (coins) that players earn by playing.

With cosmetics (frames, banners, badges, titles) instead of pay-to-win.

With a Windows anti-cheat client + server-side analysis.

With Google Cloud CS2 servers, spun up on demand to save money.

Players:

Log in with email and Steam.

Queue for 5v5 matches on Eclip servers.

Gain XP, ranks, coins, achievements, cosmetics.

Spend coins on cosmetics and possibly Steam gift cards once they reach a threshold.

Admins & moderators:

Manage users, bans, matches, cosmetics, site config.

Monitor anti-cheat events and suspicious players.

Toggle maintenance mode, update logo/banner, manage rewards.

1.2 Target Users

CS2 players who want:

Fair(er) matchmaking.

Clear progression.

Rewards for playtime (coins ‚Üí cosmetics/gift cards).

Small teams/stacked friends who want to queue together.

Streamers who want an alternative platform to FACEIT/ESEA.

1.3 Key Differentiators

Progression & Cosmetics: Levels, ranks, trophies, inventory, flashy profile.

Soft Currency & Rewards: Coins from playing, daily missions, achievements.

Anti-Cheat Transparency: Admins see AC events per match/player.

Modern UX: Mobile-friendly, responsive, attractive neon/green style.

Cost-Optimized Servers: Servers spin up only when matches exist, using existing Google Cloud credits, Neon free plan, Redis free tier, Vercel free tier, Cloudinary free tier.

Receives match created event.

Calls GCP to create CS2 VM instance using template:

Region matches player region.

Startup script sets:

Map rotation

Tickrate

Match ID

Waits until server is reachable.

Updates Match with:

serverHost, serverPort

status = LIVE or READY.

Step 3: Notify Players

Backend sends:

Notification: ‚ÄúMatch found‚Äù

Frontend shows:

Match found panel with connect info.

Or an auto-connect link string connect IP:PORT; password xyz.

Step 4: During Match

CS2 server:

Logs kills, deaths, rounds, etc.

Either:

Using plugin / extension

Or reading logs externally and posting to API

AC client:

Sends telemetry.

Step 5: Match End

Server (or plugin) calls backend:

POST /api/matches/:id/result

Includes:

Final score

Per-player stats:

Kills, deaths, assists, HS, MVPs, damage, etc.

Backend:

Sets match status = FINISHED.

Writes all stats to DB.

Updates user summary stats.

Calculates:

MMR change

Rank changes

XP & coin rewards

Enqueues notifications.

Orchestrator:

Shuts down VM so it stops burning credits.

10. ANTI-CHEAT SYSTEM DESIGN (OVERVIEW)
10.1 AC Client (Windows)

Responsibilities:

Detect presence of known cheat processes (blacklist).

Monitor suspicious overlay windows.

Collect gameplay-related metrics:

Mouse movement patterns

Reaction times

Bhop/movement patterns

Communicate securely with AC backend (HTTPS or secure WebSocket).

Only runs while:

Eclip match is active.

Player is connected to Eclip‚Äôs CS2 servers.

Does NOT:

Spy on unrelated activity.

Log keystrokes.

Capture full-screen frames (unless explicitly accepted in future).

10.2 Server-side Telemetry

CS2 server plugin / log reader will send:

Per-kill data:

Killer & victim SteamIDs

Weapon

Headshot yes/no

Distance

Round events:

Round winner

Bomb plant/defuse (later)

Match summary stats:

K/D/A

MVPs

Score

10.3 AC Backend Logic

Data stored as ACEvent records:

userId

matchId

code (e.g., KNOWN_CHEAT_PROCESS, AIM_SNAP_PATTERN, etc.)

severity (1‚Äì10)

details string

createdAt

It maintains a suspicion score for each user:

Each ACEvent adds to the score (with severity).

Score decays over time without new events.

Thresholds:

Low: visible only in admin panel.

Medium: maybe trust factor adjustments.

High: automatic queue restrictions / temp bans.

Extreme: automatic permanent bans (for blatant cheat signatures).

Integration:

AC events shown in:

User detail view (admins).

Match detail (which players flagged).

AC bans integrated with normal ban system.
